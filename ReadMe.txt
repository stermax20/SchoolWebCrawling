JAVA 학교 사이트 웹 크롤링 프로젝트 보고서


프로젝트 이름 또는 주제:

경소고 사이트 알림마당 게시판의 대외 수상, 보도 자료, 채용 공고의 정보를 크롤링하는 JAVA 프로그램 만들기.

프로젝트 주제 선정 이유:

자바 개인 프로젝트로 수업 시간에 배운 크롤링을 써야겠다고 생각했다. 왜냐하면 크롤링은 원하는 정보를 마음대로 출력할 수 있기 때문에 개인 프로젝트에 써서 활용하기 적합해 보였다. 또한 크롤링을 할 만한 사이트를 찾던 중 우리 학교 사이트가 괜찮아 보였다. 우리 학교 게시판의 정보를 크롤링하면 내게 필요한 정보를 나중에 보고 3학년 때 회사에 취업하는 데 도움이 되기 때문이다.

기대효과:

학교 게시판 크롤링을 통해 얻은 정보로 취업 소식 및 기업 정보를 활용하여 취업 전략을 효과적으로 짤 수 있다. 그리고 이번 프로젝트로 배운 자바 기술을 활용하여 다른 프로젝트를 할 때 연관 지어 기술을 사용할 수 있다. 또한 이 프로젝트를 사장하지 않고 수정하여 웹이나 프로그램으로 크롤링 결과를 출력할 수 있게 하고 다른 학교도 쓸 수 있게 상용화한다면 더욱더 많은 자바 기술과 경험을 쌓을 수 있다.

사용 기술:

JAVA(Jsoup Library + Selenium Framework), Apache Maven

주요 기능 설명:

처음 콘솔 창에 3가지 선택지로 알림마당 -> 보도자료, 알림마당 -> 대외수상, 알림마당 -> 채용공고가 뜬다. 1번을 선택하면  보도자료의 주의 사항과 함께 그 게시판의 최대 페이지 수를 표시한다. 그리고 원하는 페이지를 숫자로 입력하면 Chrome Driver로 사용자가 입력한 페이지 수까지 파싱하여 정적 + 동적 웹 크롤링을 한다. 크롤링의 결과로 게시물 번호, 게시물 제목, 작성자, 조회수, 등록일이 표시되고 한 페이지가 끝날 때마다 끝부분에 현재 보도자료 페이지 번호를 표시한다.

프로젝트 진행 시 어려웠던 점:

처음에는 JAVA에서 Jsoup Library만을 이용한 정적 웹 크롤링 코드를 작성하였다. 하지만 학교 사이트가 디자인이 변경되었다. 따라서 게시판의 각 페이지 URI을 반복문으로 처리하는 정적 방식의 코드에 버그가 생겼다. 사용자가 고른 게시판의 첫 페이지만 입력한 페이지 수만큼 반복되어서 출력되는 버그였다.

난관 극복 방법 및 과정:

버그를 고치기 위해 개발자 도구로 학교 사이트의 페이지 처리 부분을 분석하였다. 분석한 결과 페이지 버튼을 누를때 Javascript로 onClick으로 goPaging() Event를 발생시켜 페이지를 넘긴다는 것을 알았다. 그래서 동적 웹 크롤링을 하기 위해 이클립스 IDE에 Apache Maven 프로젝트를 생성하였다. 그리고 기존의 Jsoup Library를 이용한 코드에 페이지 처리 부분을 Selenium Framework를 사용하여 Chrome Driver로 자동화하여 동적으로 처리하였다. 모두 동적으로만 처리할 경우 기존의 정적 코드보다 속도가 많이 느려져서 정적과 동적을 함께 사용하였다.
그 결과 성공적으로 페이지 처리 부분이 자동화 되어 학교 사이트가 정적 + 동적 웹 크롤링이 되는 것을 확인할 수 있었다.  

개인 프로젝트 완성 과정에서 배운 점:

생각보다 각 웹 사이트의 CSS 요소 & JavaScript 함수 부분을 분석하여 크롤링하는 것이 복잡하다는 것을 느꼈다. 왜냐하면 웹 사이트는 자주 변화한다는 것을 몸소 느꼈기 때문이다. 그리고 JAVA의 여러 라이브러리와 프레임워크를 사용하면 다양한 동작을 수행할 수 있다는 것을 알았다. 또한 동적 웹 크롤링은 JavaScript 처리 부분에는 유용하지만, 처리 속도가 느리고 정적 웹 크롤링은 처리 속도가 빠르지만, 동적인 부분을 처리 못 한다는 것을 알았다. 이 두 가지 방법을 적절히 조합하여 사용하면 효율적으로 웹 크롤링을 수행할 수 있다는 것을 느꼈다. 마지막으로 웹페이지의 동작을 분석하고 문제를 해결하기 위해 개발자 도구를 사용하였다. 이를 통해 버그를 발견하고 이를 수정하는 능력을 기를 수 있었다.

현재 관심있는 분야 및 미래 진로 방향:

내가 현재 관심 있는 분야는 DataBase이다. 차후 Oracle DB, MySQL 등을 공부하여 DBA가 되고   싶다. DB는 구조화된 데이터를 저장, 검색, 수정 및 삭제하는 데 사용된다. Backend는 이러한 DB를 효율적으로 관리하고 상호 작용할 수 있는 인터페이스를 제공한다. 전공 수업인 JAVA를 더 공부하여 Spring, Spring Boot 같은 Backend 처리를 공부하고 미래에 DBA로서 DB에 JDBC로 연결도 하는 개발자가 되고 싶다.
